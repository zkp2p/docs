## Summary

- Updates the seller "Provide Liquidity" guide to reflect the PayPal hard cutover from email to PayPal.me username as maker identity
- Changes payee-details list entry from "PayPal Email" to "PayPal.me Username" with usage example
- Bumps PeerAuth extension minimum version from v0.4.13+ to v0.4.14+ in the admonition
- Adds explanation that maker identity is now keyed off normalized PayPal.me username

## Why

zkp2p-clients PR #694 (merged 2026-04-23) shipped a breaking change: PayPal maker deposits now use `normalizePaypalMeUsername(payeeDetails)` as `offchainId` instead of email, and `minExtensionVersion` was bumped to `0.4.14`. The seller guide still referenced "PayPal Email" and "v0.4.13+", which would cause new sellers to enter the wrong identity and hit a silent registration failure.

Docs PR #68 already fixed the developer-facing equivalent in `developer/offramp-integration.md`. This handles the seller-facing user guide.
